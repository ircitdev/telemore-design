1. Что такое OpenPencil

OpenPencil — open-source векторный дизайн-инструмент (альтернатива Figma / Sketch / Pencil.dev).

Главные особенности для нас:

  • 📄 Файлы .op — это JSON-текст, отлично работают с Git (видно дифф изменений)
  • 🌐 Есть веб-версия — можно работать прямо в браузере без установки
  • 🤖 Есть AI-ассистент для генерации UI из промптов
  • 🔧 Есть CLI op для автоматизации (нам уже пригодился — мы сгенерили wireframes скриптом)

2. Два способа работы

🌐 Способ A: Браузер (быстрый старт)

  • Открой https://design.telemore.org
  • Файл telemore.op уже там, в /opt/telemore/designs/
  • ✅ Плюс: ничего ставить не надо
  • ⚠️ Минус: только один человек редактирует за раз (последний сохранивший перезапишет чужие правки)
  • 👉 Подходит для: быстрых правок, демо клиенту, просмотра

💻 Способ B: Desktop + Git (рекомендуем для дизайнеров)

  • Ставишь приложение на свой Windows/Mac/Linux
  • Клонируешь Git-репо проекта
  • Работаешь в своей Git-ветке (никто никому не мешает)
  • Делаешь pull / merge через визуальный three-way merge внутри OpenPencil
  • ✅ Плюс: параллельная работа, история, откаты, безопасно
  • 👉 Подходит для: основной работы над проектом

Договорённость команды: правки больше 5 минут — только через desktop + git. Браузер — только для просмотра и крошечных фиксов.

3. Установка Desktop

Windows

  1. Скачай OpenPencil-0.7.4-x64-win.exe со страницы Releases
  2. Положи куда удобно (например, D:\Apps\OpenPencil\)
  3. Двойной клик — запустится без установки

Альтернатива через Scoop:

scoop bucket add openpencil https://github.com/zseven-w/scoop-openpencil
scoop install openpencil

macOS

brew tap zseven-w/openpencil
brew install --cask openpencil

Linux

  • Скачай .deb или .AppImage из Releases
  • Для AppImage: chmod +x OpenPencil-*.AppImage && ./OpenPencil-*.AppImage

CLI op (опционально, нужен для скриптов)

npm install -g @zseven-w/openpencil

Проверка: op --version → 0.7.4

7. Ежедневная работа

Утро: получить свежие изменения

cd telemore-design
git checkout main
git pull

Или в OpenPencil: Git → Pull

Перед началом задачи: сделать ветку

В OpenPencil: Git → New Branch → имя ветки

Правила именования:

  • feat/screen-rating-tabs — новая фича/экран
  • fix/qr-pdf-export — починка
  • tweak/colors-vice-city — мелкие правки стиля
  • wireframe/admin-A11-stats — новый wireframe админки

Работа: правки в файле

  1. Перейди на нужную Page слева
  2. Меняй элементы (двойной клик по тексту → редактирование)
  3. Cmd+S / Ctrl+S — сохранить (часто! каждые 5-10 минут)
  4. Если правка большая — делай промежуточные коммиты

После каждой «логической части» — коммит

В OpenPencil: Git панель → Commit Стандарт сообщения:

<scope>: <что сделано>

screen-qr: увеличить QR до 240px и добавить рамку
admin-A3: добавить колонку «Дата регистрации»
fix: поправить fontFamily на кнопке ШТРАФ
wireframe/A6: добавить опцию «Повтор еженедельно»

Завершил задачу — push + PR

  1. git push (или Git → Push в OpenPencil)
  2. Открой репо в GitHub/GitLab → создай Pull Request в main
  3. Получи аппрув от тимлида → merge

10. Правила командной работы

✅ Делай

  • 🌿 Всегда работай в отдельной ветке (никогда напрямую в main)
  • 💾 Сохраняй часто (Ctrl+S каждые 5-10 минут)
  • 📝 Делай частые коммиты с понятными сообщениями
  • 🔄 Перед началом дня — git pull
  • 👀 Смотри what changed перед коммитом (Git → Diff)
  • 🎨 Соблюдай палитру и шрифты — это узнаваемость бренда
  • 💬 В сложных правках — пиши в командный чат, что меняешь и зачем

❌ Не делай

  • 🚫 Не переименовывай ID нод в коде (если знаешь, как залезать в код .op) — это сломает другие правки в merge
  • 🚫 Не редактируй через браузер, если знаешь, что коллега сейчас работает в desktop — потеряете работу
  • 🚫 Не коммить «WIP» куски, которые ломают экран — коммитим только рабочее
  • 🚫 Не используй Pricedown для русского текста, если шрифт БЕЗ Cyrillic-версии — будет квадратами
  • 🚫 Не добавляй экраны без согласования — лучше создай ветку и обсуди
  • 🚫 Не удаляй чужие страницы (Pages) — даже если кажется, что они «лишние»

11. Типовые задачи

Задача: исправить опечатку в тексте

  1. Открой нужную Page
  2. Двойной клик по тексту → правь
  3. Ctrl+S → Git → Commit "fix: опечатка в названии ачивки" → Push

Задача: поменять цвет кнопки

  1. Выдели кнопку (фрейм)
  2. Справа в панели Properties → Fill → измени HEX
  3. Если есть stroke (рамка) — поменяй и там
  4. Сохрани, закоммить

Задача: добавить новый экран мобилки

  1. Слева в Pages → правый клик → New Page → имя «8. Новый экран»
  2. На canvas: создай root frame 375 × 812 с фоном-градиентом из палитры
  3. Внутри собери UI из существующих компонентов (можно скопировать Ctrl+C из соседнего экрана)
  4. Соблюдай отступы / шрифты / glow-стиль
  5. Коммит feat: добавить экран Настройки

Задача: экспортировать макет для презентации клиенту

  1. Выдели нужный root frame на canvas
  2. Ctrl+Shift+P → выбери формат (PNG / JPG / PDF) → размер 1x / 2x / 3x
  3. Save

Задача: дать ссылку клиенту на просмотр

  • Отправь https://design.telemore.org — клиент откроет в браузере, увидит все макеты

Задача: разрешить merge-конфликт

  1. После git pull если есть конфликт — OpenPencil откроет Conflict Resolution Panel
  2. По каждой ноде: выбери Mine / Theirs / Both
  3. Иногда нужен ручной правка — кликни по узлу, отредактируй
  4. Когда все конфликты решены → Mark Resolved → Commit
  5. Push

12. AI в OpenPencil (Gemini/Claude/GPT)

OpenPencil умеет генерировать дизайны из промптов и модифицировать существующие (выделил → попросил → агент переделал). Всё это через встроенный AI-чат (Ctrl+J / Cmd+J).

🎯 Главное: что у нас настроено

Что Как работает Для кого настраивать
Gemini CLI (серверная настройка) ✅ АКТИВНО Ключ лежит на VPS в Docker env (/etc/telemore/openpencil.env). Запущено через systemd-unit telemore-openpencil.service. «Просто работает» для всех. Доступны модели Gemini 3 Pro, 3 Flash, 3.1 Pro/Flash Настроено тимлидом
Built-in Provider (Google Gemini) Ключ хранится в браузере каждого пользователя (localStorage) Настраивается каждым лично, если хочется выбрать конкретную модель
Claude Code / Codex / Copilot / OpenCode Локальные CLI на твоей машине через авторизацию этих продуктов Только desktop-версия OpenPencil

🅰️ Использовать Gemini CLI (самое простое, ничего не настраивать)

  1. В OpenPencil: Cmd+, / Ctrl+, → вкладка Agents
  2. В секции AI Agents найди Gemini CLI → должно быть «Connected via API key»
  3. В AI-чате (Cmd+J) выбери агента Gemini CLI → модель
  4. Пиши запрос (например: «нарисуй экран настроек в стиле Vice City с переключателем темы»)

Если «Not connected» — скажи тимлиду, проверит env var на VPS.

🅱️ Использовать собственный ключ (Built-in Provider)

Если хочешь работать со своим ключом Gemini / OpenAI / Anthropic:

  1. Получи ключ
    • Gemini: https://aistudio.google.com/apikey (бесплатный лимит 1500 req/day)
    • OpenAI: https://platform.openai.com/api-keys (платный)
    • Anthropic: https://console.anthropic.com/settings/keys (платный)
  2. OpenPencil → Cmd+, → Agents → Встроенные провайдеры → Добавить провайдера
  3. Выбери провайдера, вставь ключ, нажми Добавить
  4. В AI-чате выбирай нужного провайдера и модель

⚠️ Безопасность:

  • Твой ключ хранится только в твоём браузере (localStorage), никуда не отправляется
  • НЕ шарь скриншоты с открытым ключом
  • НЕ коммить ключи в репозиторий

🎨 Что умеет AI в OpenPencil

  • Prompt-to-Canvas — из текста генерится полноценный UI на канвасе
  • Orchestrator — разбивает большие запросы на секции, параллельно рисует
  • Agent Teams — несколько агентов работают одновременно над разными блоками
  • Модификация выделения — выделил элементы → «Сделай кнопки неоновыми»
  • Vision input — прицепи скриншот/референс → «Нарисуй похоже, но в нашем стиле»
  • Style Guides — применение готовых стилей (glassmorphism, brutalist, retro и т.д.)

⚙️ Инфраструктура Gemini на нашем сервере

Ключ живёт в /etc/telemore/openpencil.env на VPS (права 600, только root). Контейнер запускается systemd-юнитом telemore-openpencil.service с флагом --env-file.

Как заменить/ротировать ключ (только тимлид):

ssh root@2.27.4.90
sudo nano /etc/telemore/openpencil.env    # замени значение
sudo systemctl restart telemore-openpencil.service

Проверка статуса:

ssh root@2.27.4.90 'systemctl status telemore-openpencil.service --no-pager'
ssh root@2.27.4.90 'docker logs --tail 20 openpencil'

Тест Gemini с клиентской стороны:

curl -s -X POST https://design.telemore.org/api/ai/connect-agent \
  -H "Content-Type: application/json" \
  -d '{"agent":"gemini-cli"}' | head -c 200
# Ожидаемый ответ: {"connected":true,"models":[...]}

Ключ никогда не коммитится в репо. Лежит только на VPS. Команда пользуется Gemini через «Gemini CLI» agent в OpenPencil без настройки.

13. Автосинхронизация

Тебе не нужно вручную деплоить ничего на сервер. Всё происходит само:

Кто-то делает git push в main
        ↓
GitHub Actions (~9 секунд)
        ↓
SSH в VPS → git pull --hard origin/main
        ↓
Cron на VPS (каждые 2 мин) — fallback
        ↓
Обновляется:
 • https://design.telemore.org/files/* (все файлы репо)
 • https://design.telemore.org/manual/ (этот гид)
 • https://design.telemore.org/go/ (launcher)
 • https://app.telemore.org/ (coming-soon)

Что это означает для тебя

Действие Что обновится Когда
Push коммита с telemore.op Файл на /files/telemore.op Через ~9 сек
Push изменения в TEAM_GUIDE.md + пересборка manual/index.html Этот HTML-гид Через ~9 сек
Push изменения deployed-app/index.html https://app.telemore.org/ Через ~9 сек

Если push сделан, но веб не обновился

  1. Проверь GitHub Actions: https://github.com/ircitdev/telemore-design/actions — последний run должен быть зелёный
  2. Если красный — открой run, читай логи, попроси тимлида
  3. Жди до 2 минут — сработает cron-fallback на VPS
  4. Если всё равно не обновилось — пиши тимлиду

Ручной триггер деплоя

gh workflow run deploy.yml --repo ircitdev/telemore-design

Проверка статуса с командной строки

# Логи деплоя на VPS
ssh root@2.27.4.90 'tail -20 /var/log/telemore-sync.log'

# Состояние контейнеров
ssh root@2.27.4.90 'docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"'

14. Troubleshooting

Кириллица отображается квадратами 🟦🟦🟦

Причина: не установлен Pricedown с кириллицей или Roboto. Решение: см. раздел 4. Установка шрифтов. После установки — перезапусти OpenPencil.

Файл telemore.op повреждён / не открывается

Решение:

cd telemore-design
git checkout main
git pull
git reset --hard origin/main   # ⚠️ удалит локальные несохранённые правки

Затем открой telemore.op заново.

OpenPencil не видит .op файл при открытии

Решение: перезапусти приложение → File → Open Recent / Open File → выбери telemore.op вручную.

Браузерная версия пишет «не удалось загрузить»

Решение: обнови страницу (Cmd+Shift+R / Ctrl+Shift+R), очисти кэш. Если всё равно — пиши в чат, перезапустим контейнер на сервере.

Конфликт в Git, не понимаю как решить

Решение: не паникуй, не делай git reset --hard. Пиши в командный чат, тимлид поможет. В крайнем случае — сохрани свою ветку под другим именем:

git stash                 # отложить незакоммиченные правки
git checkout -b backup-myname-$(date +%F)
git push -u origin backup-myname-$(date +%F)

Потом разберёмся вместе.

Случайно удалил страницу / экран

Решение:

  1. Если ещё не сохранил → Ctrl+Z (undo)
  2. Если сохранил, но ещё не закоммитил → git restore telemore.op
  3. Если закоммитил → откатить через git revert <hash коммита>

Что-то странное — не работает фича OpenPencil

  • Проверь, последняя ли у тебя версия (Help → Check for Updates)
  • Перезапусти приложение
  • Проверь логи: Help → Show Logs
  • Если баг — кидай скриншот в чат и issue в репозитории OpenPencil

Я открыл /go/, но OpenPencil не открылся в новой вкладке

Браузер заблокировал popup. На /go/ появится кнопка «Открыть OpenPencil →» — нажми её. Файл уже в Downloads.

Я залил коммит, но на https://design.telemore.org/files/telemore.op старая версия

  • Проверь GitHub Actions: https://github.com/ircitdev/telemore-design/actions
  • Если зелёный — обнови вкладку с Ctrl+Shift+R (hard reload, чистит кэш)
  • Cloudflare иногда кэширует на 5-30 секунд

Браузерная версия пишет «Не получается прочитать файл .op»

Скорее всего ты выбрал устаревший / битый файл из Downloads. Скачай свежую копию через https://design.telemore.org/go/ или https://design.telemore.org/files/telemore.op
