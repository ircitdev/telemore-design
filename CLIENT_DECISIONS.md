## 2. Рекомендация по VPS

Для production-ready запуска (60 пользователей, 13 дней) минимум:

| Ресурс | Минимум | Рекомендую |
|--------|---------|-----------|
| vCPU | 2 | **2-4** |
| RAM | 4 GB | **4-8 GB** |
| SSD | 40 GB | **60-80 GB** (на бэкапы и логи) |
| Network | 100 Mbps | 1 Gbps |
| OS | Ubuntu 24.04 LTS | как сейчас |

**Что будет на VPS** (продакшн):
- PostgreSQL 16 (БД)
- Node.js + Fastify (API + бот)
- Redis (опционально, для rate limiting / очередей)
- Nginx (reverse proxy)
- Frontend (Next.js, статика отдаётся nginx)
- Cloudflare Tunnel (SSL без открытия портов)
- Бэкапы pg_dump через cron → R2/S3

**Текущий сервер** (`2.27.4.90`): Ubuntu 24.04, на нём уже работает OpenPencil (дизайн-инструмент) + landing app.telemore.org. Проверить характеристики:

```bash
ssh root@2.27.4.90 'nproc && free -h && df -h /'
```

Если меньше 4GB RAM или 40GB диска — нужно проапгрейдить тариф у хостера до запуска бэкенда.

## 4. Уточнения для БД/API (новые требования)

### Новые поля в таблицах

```sql
-- transactions: возможность отката
ALTER TABLE transactions ADD COLUMN reversed_at TIMESTAMPTZ;
ALTER TABLE transactions ADD COLUMN reversed_by UUID REFERENCES users(id);
ALTER TABLE transactions ADD COLUMN original_transaction_id UUID REFERENCES transactions(id);

-- tours: chat_id для поздравлений
ALTER TABLE tours ADD COLUMN birthday_chat_id BIGINT;
ALTER TABLE tours ADD COLUMN timezone VARCHAR(50) DEFAULT 'Europe/Moscow';

-- tour_participants: история смены отряда
CREATE TABLE squad_changes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  participant_id UUID REFERENCES tour_participants(id),
  from_squad VARCHAR(100),
  to_squad VARCHAR(100),
  changed_by UUID REFERENCES users(id),
  reason TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- events: возможность авто-начисления
ALTER TABLE events ADD COLUMN auto_xp_on_checkin INT DEFAULT 0;
ALTER TABLE events ADD COLUMN bulk_mode BOOLEAN DEFAULT false;
```

### Новые API endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `POST` | `/api/transactions/:id/reverse` | Откат транзакции (с указанием причины) |
| `PATCH` | `/api/participants/:id/squad` | Сменить отряд участника |
| `POST` | `/api/scanner/bulk-session/start` | Начать bulk-режим: фиксированная сумма + событие |
| `POST` | `/api/scanner/bulk-session/scan` | Скан в bulk-режиме (одинаковая сумма всем) |
| `POST` | `/api/scanner/bulk-session/finish` | Завершить bulk-сессию + вернуть статистику |

## 6. Ответы на уточняющие вопросы (v3)

| # | Вопрос | Ответ клиента | Реализация |
|---|--------|---------------|-----------|
| 1 | Срок отката транзакций | **24 часа** от создания | В API — проверка `created_at > now() - INTERVAL '24 hours'` |
| 2 | Кто откатывает | **Автор транзакции + суперадмин** | В API — `created_by = current_user.id OR role = 'superadmin'` |
| 3 | Bulk-сканирование: дубли | **Игнорируем в рамках одной сессии**. В разных сессиях один участник может получить баллы за несколько активностей | `UNIQUE(session_id, participant_id)` — внутри сессии уникально. Отдельные сессии — отдельные записи |
| 4 | Bulk-сессия: автозавершение | **Через 30 минут бездействия + ручная кнопка** | Cron или событие при неактивности; UI — кнопка «Завершить» |
| 5 | Баланс при смене отряда | **Остаётся у участника** | Не трогаем `balance`; в `squad_changes` просто фиксируем факт смены |
| 6 | Кто меняет отряд | **Продюсер + суперадмин** (участник НЕ может сам) | API — проверка `role IN ('producer','superadmin')` |
| 7 | PDF бейдж | **Только QR 4×4 см** от API. Макет бейджа 10×15 — отдельный функционал (см. ниже §8) | `GET /api/participants/:id/qr.png` — 400×400 px |
| 8 | Лого TELEMORE | **Генерируем варианты в стиле GTA Vice City** | Задача тимлида — накидать 3–5 вариантов через Gemini/Midjourney |
| 9 | Список ачивок | **Клиент подготовит сам** с дизайном | Ждём список, до этого — заглушки по шагу 100 XP |

## 7. Новое требование: Лидерборд «топ + моё место»

**Референс:** https://design.telemore.org/files/... (скриншот МЭШ от клиента — видны первые 10 и место владельца аккаунта).

### Логика
- Отображается **топ-N** участников (по умолчанию **10**)
- Если пользователь **вне топа** — его место показывается отдельно внизу **после разделителя «• • •»**
- **Анонимизация** (опционально): участники вне топа показываются как «Белочка», «Котик» и т.п. + эмодзи-аватарка (как у МЭШ)
- Настройка в супер-админ панели per-tour: **10 / 20 / все** + переключатель анонимности

### Новые поля в `tours.config`

```json
{
  "leaderboard": {
    "visible_count": 10,
    "anonymize_below_top": true,
    "anonymous_nicknames": ["Белочка", "Котик", "Лисичка", "Пингвин", "Ёжик", "..."]
  }
}
```

### Экраны
- **screen-5 (Рейтинг)** — переверстан: топ-10 → разделитель → подсвеченная карточка «Я». Бронза/серебро/золото у первых трёх. Переключатели «Личный/Команда» и «День/Тур».
- **A10 (Супер-админ)** — добавлена секция «Настройки рейтинга».

## 8. Новое требование: Генератор бейджей 10×15

### Идея клиента
> На анкету можно загрузить одну фотку, загружает продюсер, чтобы на ней было видно крупно лицо. И на бейдже размером 10×15 автоматически указывается фото, Телеморе, название смены, имя пользователя и его QR. При этом используется стилистика смены.

### Реализация

| Компонент | Что |
|-----------|-----|
| **Загрузка фото** | Продюсер в карточке участника (A4) — прямоугольник с кнопкой «Загрузить / Заменить». Хранится в Supabase Storage / S3 |
| **PDF генератор** | Серверный сервис (Node.js + puppeteer / pdf-lib). Берёт HTML-шаблон 10×15 с брендингом смены, подставляет данные, рендерит в PDF |
| **Пакетный режим** | A11 (новая страница админки) — «Генератор бейджей». Чекбоксы по участникам → PDF пакет для печати |
| **Единичный режим** | Из карточки участника (A4) — кнопка «PDF бейдж» |
| **Размер** | A6 landscape (10×15 см); внутри: фото 6×6 см + лого + название смены + имя + QR 4×4 см |
| **Пресет бейджа** | В `tours.config.badge_template` — фон, шрифт, рамка, под стилистику смены |

### Новые поля в БД

```sql
ALTER TABLE users ADD COLUMN photo_url TEXT;
ALTER TABLE users ADD COLUMN photo_uploaded_by UUID REFERENCES users(id);
ALTER TABLE users ADD COLUMN photo_uploaded_at TIMESTAMPTZ;

-- tours.config JSON получает поле badge_template
```

### Новые API endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `POST` | `/api/participants/:id/photo` | Загрузка фото (multipart) — только продюсер/суперадмин |
| `DELETE` | `/api/participants/:id/photo` | Удаление фото |
| `GET` | `/api/participants/:id/badge.pdf` | PDF бейдж 10×15 |
| `POST` | `/api/badges/bulk.pdf` | Пакет бейджей (массив `participant_ids`) → многостраничный PDF |
